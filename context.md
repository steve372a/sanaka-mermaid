# Context

## 这是什么

这是一个自用的本地网页工具：用拖拽方式生成 Mermaid `flowchart TD` 图表。

- 运行方式：`npm start`
- 本地服务：Node.js 原生 `http`，入口是 `server.js`
- 前端页面：`public/index.html`
- 样式：`public/styles.css`
- 交互与状态：`public/app.js`

## 目标

提供一个不依赖复杂后端的本地可视化编辑器，核心能力是：

- 从左侧节点库拖入节点
- 在画布中拖动节点
- 从输出口拖到输入口创建连线
- 自动生成 Mermaid 代码
- 右侧实时渲染预览
- 支持本地自动保存、JSON 导入导出
- 支持导出 SVG / PNG
- 支持修改 Mermaid 代码并应用回画布

## 已做改动

### 初始实现

- 建了一个最小 Node.js 项目，不依赖打包器
- 用浏览器端 Mermaid CDN 做实时渲染
- 实现了基础节点类型：
  - `rect`
  - `round`
  - `diamond`
  - `circle`
  - `subroutine`
- 加了本地存储 `localStorage`
- 加了复制 Mermaid 代码、导入/导出 JSON

### 拖拽修复

之前拖入画布有概率无效，原因是原实现只依赖 `dataTransfer["application/json"]`，浏览器兼容性不稳。

已修复为：

- 同时写入 `application/json` 和 `text/plain`
- 增加内存态兜底 `state.paletteDrag`
- `dragover` 时显式设置 `dropEffect = "copy"`
- 增加点击左侧节点直接插入，避免原生拖放失灵时完全不可用

### 删除节点修复

之前节点删除和 `pointerdown` 里的选中、拖动、连线走同一条事件链，存在偶发冲突。

已修复为：

- 删除按钮从 `pointerdown` 流程中拆出
- 改为单独的 `click` 事件处理删除
- `pointerdown` 遇到删除按钮时直接跳过
- 增加键盘 `Delete` / `Backspace` 删除当前选中节点或连线

### 导出和代码编辑

已新增：

- 导出当前预览为 `SVG`
- 导出当前预览为 `PNG`
- 右侧代码框可直接编辑
- “应用代码”会把代码重新解析回画布

### 代码删除修复

之前“应用代码”时，边里的 plain id 会隐式补回节点，所以你删掉节点声明但没删到对应边时，节点会再次出现，看起来像“删不掉”。

已修复为：

- 如果代码里存在独立节点声明行，就把这些声明视为节点存在性的权威来源
- 删除节点声明后，关联的悬空边会一起被丢弃，不再隐式补节点
- 支持“空代码应用 = 清空画布”

### 代码与画布状态同步修复

之前右侧代码改完如果没有先点“应用代码”，再去画布里拖新节点、删节点、连线，系统仍然会基于旧画布状态继续操作，于是旧节点会被重新写回，看起来像“删了又回来”。

已修复为：

- 代码一旦改动，任何画布侧变更前都会先自动应用当前代码
- 如果代码解析失败，会阻止后续画布操作
- 这样代码和画布不会再分叉

### 刷新清空

用户要求页面刷新后不保留任何内容。

已改为：

- 启动时清掉旧 `localStorage`
- 刷新页面后从空画布开始
- 不再使用本地自动持久化

当前代码回写画布只支持本工具这一套 Mermaid `flowchart` 子集：

- `flowchart TD`
- `flowchart BT`
- `flowchart LR`
- `flowchart RL`
- 节点形状：
  - `["矩形"]`
  - `("圆角")`
  - `{"判断"}`
  - `(("圆形"))`
  - `[["子流程"]]`
- 连线：
  - `A --> B`
  - `A -->|标签| B`

## 当前交互规则

- 左侧节点库：
  - 可拖到画布
  - 也可直接点击插入到当前可视区域附近
- 画布节点：
  - 按住主体拖动位置
  - 拖右侧端口到其他节点左侧端口可创建连线
  - 点击右上角 `×` 删除节点
- 右侧 Inspector：
  - 选中节点后可改标题和形状
  - 选中连线后可改连线标签
- 右侧代码框：
  - 可直接修改 Mermaid 代码
  - 修改后会实时重新预览
  - 点击“应用代码”后同步回画布
  - 即使不手动点应用，只要开始做画布侧改动，也会先自动应用当前代码
- 键盘：
  - `Delete` / `Backspace` 删除当前选中项

## 注意点

- Mermaid 预览依赖 CDN；如果要完全离线，需要把 Mermaid 改成本地依赖
- 目前代码回写画布主要支持 `flowchart` 子集，还没做更完整的 Mermaid 语法解析
- 如果代码混用“独立节点声明”和“仅在边里出现的 plain id”，应用时会优先按独立声明来裁剪节点集合
- 还没做序列图、类图、状态图

## 如果下次继续改

优先顺序建议：

1. 把左侧节点拖放改成完全自定义的 pointer-based 拖放，进一步摆脱原生 DnD 兼容性问题
2. 给画布加缩放和平移
3. 增加边的箭头类型、虚线、方向配置
4. 扩展 Mermaid 代码解析，支持更多语法
